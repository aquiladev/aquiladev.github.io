nHook - Continuous delivery server for NuGet packages

If you want to share your package immediately after making changes in your solution, this service is for you. nH is continuous delivery server for NuGet packages. The service automatically triggers the build and publishes your package into NuGet when you push changes into your version control system.

NuGet packages

The service builds packages according to supported types of NuGet. In order to publish your packages you should provide ApiKey, that you can find in your NuGet account. After publishing your package is going to be immediately available in NuGet, and you can manage your packages through https://nuget.org/account/packages
*If you don't know what is NuGet, you can read documentation.

Getting started

1. Put nuspec file next to your project file. If you don't have a project file, put nuspec in the root of your repository.
2. Set up your version control system.

Version Control Systems

  • GitHub

    You can use several options for GitHub:
    • Webhook
      You should open repository's Setting > Webhook&Services > Add webhook and fill form:
      Payload URL: http://api.nhook.net/github/{NuGetApiKey}
      Content type: application/json
    • Service (recommend)
      You should open repository's Setting > Webhook&Services > Add service, select NHook and write ApiKey in the appropriate field.

  • BitBucket

    You should open repository's Setting page to set up BitBucket repository > Hooks, select POST option and write URL: http://api.nhook.net/bitbucket/{NuGetApiKey}
